# ReproRun

> Дайте ему статью. Он скажет вам, действительно ли результаты воспроизводятся.

**ReproRun** — это пользовательский навык [Claude Code](https://claude.com/claude-code), который автоматизирует мучительный путь от *«в статье утверждается X»* до *«X действительно работает на моей машине»*. Статьи публикуют красивые цифры; их воспроизведение обычно умирает на сломанном коде, несобираемых окружениях и расхождении версий. ReproRun берёт на себя весь конвейер от начала до конца:

**прочитать статью → найти код и данные → собрать окружение → провести дымовой тест → выполнить полный запуск → сравнить измеренные цифры с утверждениями статьи.**

---

## ✨ Создан для адаптации — на любой платформе

ReproRun спроектирован так, чтобы **подстраиваться под то, что ему дают**, а не предполагать одну фиксированную конфигурацию:

- **Кроссплатформенность** — работает на Windows, macOS и Linux
- **Поддержка разных языков** — полный конвейер для Python; минимальный путь для R / MATLAB / Julia
- **Поддержка разных областей** — биология одиночных клеток, машинное обучение для изображений и другое, без перенастройки под каждую область
- **Поддержка разных режимов** — *режим инструмента* (`pip install` + написание вызывающего скрипта) или *режим эксперимента* (клонирование репозитория и запуск его скриптов) — выбирается автоматически для каждой статьи

---

## 🚀 Что он делает

- **Находит статью только по её названию** — автоматический поиск в интернете PDF и репозитория кода
- **Автоматически диагностирует деградацию окружения** — конфликты ABI numpy, изменения API torchvision, устаревшие методы pandas… обнаруживаются и исправляются автоматически
- **Никаких догадок по параметрам** — проверяет сигнатуры функций после установки
- **Цикл самоисцеления зависимостей из 5 раундов** — классификация ошибки → целевое исправление → повторная проверка, до 5 раундов
- **Изоляция статей** — каждая статья получает собственное пространство имён для вывода

## 🏗️ Архитектура

Один оркестратор (`SKILL.md`) управляет 6 специализированными агентами:

| Агент | Роль |
|-------|------|
| A · paper-reader | Извлекает числовые утверждения для воспроизведения |
| B · resource-finder | Находит репозиторий кода и наборы данных |
| C · environment-builder | Собирает и чинит среду выполнения (самый сложный) |
| D · smoke-tester | Быстрый дымовой тест — подтверждает, что всё запускается |
| E · full-runner | Полный запуск воспроизведения |
| F · result-comparator | Сравнивает измеренное с заявленным, пункт за пунктом |

## ✅ Валидация

Проверен от начала до конца на 4 статьях из разных областей и на разных языках:

| Статья | Область | Язык | Дымовой тест |
|-------|--------|----------|:--:|
| scVelo (Nat Biotech 2020) | одиночные клетки | Python | ✓ |
| ScType (Nat Comms 2022) | одиночные клетки | R | ✓ |
| Annotatability (Nat Comp Sci 2024) | одиночные клетки | Python | ✓ |
| Robust Stitching (ICML 2023) | ML для изображений | Python | ✓ |

**Доля прохождения дымовых тестов 4/4 · Блокеров P1 0 · 18 проверенных улучшений**

## 📦 Начало работы

ReproRun — это навык Claude Code. Чтобы его использовать:

1. Установите [Claude Code](https://claude.com/claude-code).
2. Поместите папку `paper-reproduction/` туда, где Claude Code может загружать навыки.
3. В сессии Claude Code просто попросите — например, *«воспроизведи scVelo»* или *«воспроизведи Таблицу 2 из этой статьи»*. Навык запускается автоматически.
