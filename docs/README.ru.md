# ReproRun

> Дайте ему статью. Он скажет вам, воспроизводятся ли результаты на самом деле.

**ReproRun** — это переносимый **навык AI-агента** (AI-agent skill), пригодный для использования любым совместимым AI-ассистентом для написания кода. Он автоматизирует мучительный путь от *«в статье утверждается X»* до *«X действительно запускается на моей машине»*. Статьи приводят красивые цифры; воспроизведение обычно умирает на сломанном коде, несобираемом окружении и расхождении версий. ReproRun берёт на себя весь конвейер от начала до конца:

**чтение статьи → поиск кода и данных → сборка окружения → дымовой тест → полный запуск → сравнение измеренных цифр с утверждениями статьи.**

---

## ✨ Создан адаптироваться — на любой платформе

ReproRun спроектирован так, чтобы **подстраиваться под то, что ему дали**, а не предполагать одну фиксированную конфигурацию:

- **Кросс-ОС** — работает на Windows, macOS и Linux
- **Кросс-язык** — полный конвейер для Python; минимальный путь для R / MATLAB / Julia
- **Кросс-домен** — биология единичных клеток, ML для изображений и многое другое, без перенастройки под каждый домен
- **Кросс-режим** — *режим инструмента* (`pip install` + написание вызывающего скрипта) или *режим эксперимента* (клонирование репозитория и запуск его скриптов) — выбирается автоматически для каждой статьи

---

## 🚀 Что он делает

- **Находит статью по одному только заголовку** — автоматический веб-поиск PDF и репозитория с кодом
- **Автодиагностика устаревания окружения** — конфликты ABI numpy, изменения API torchvision, устаревшие методы pandas… обнаруживаются и исправляются автоматически
- **Никаких угадываний параметров** — после установки проверяет сигнатуры функций
- **Цикл самовосстановления зависимостей из 5 раундов** — классификация ошибки → точечное исправление → повторная проверка, до 5 раундов
- **Изоляция статей** — каждая статья получает собственное пространство имён для вывода

---

## 🏗️ Архитектура

Один оркестратор (`SKILL.md`) управляет 6 специализированными агентами:

| Агент | Роль |
|-------|------|
| A · paper-reader | Извлекает численные утверждения для воспроизведения |
| B · resource-finder | Находит репозиторий с кодом и наборы данных |
| C · environment-builder | Собирает и чинит окружение выполнения (самый сложный) |
| D · smoke-tester | Быстрый дымовой тест — подтверждает, что всё запускается |
| E · full-runner | Полный запуск воспроизведения |
| F · result-comparator | Сравнивает измеренное с заявленным, пункт за пунктом |

## ✅ Валидация

ReproRun был запущен от начала до конца на реальных статьях из разных доменов. Он не просто штампует «успех» — для каждой статьи он возвращает **честный вердикт**: цифры воспроизведены, конвейер воспроизведён или *невозможно воспроизвести в текущем виде* — всегда с указанием первопричины.

| Статья | Домен | Результат |
|-------|--------|--------|
| **UMAP** (McInnes 2018, JOSS) | снижение размерности | ✅ **Цифры воспроизведены** — 11/14 метрик точности k-NN совпадают в пределах ±0.01; MNIST и Fashion-MNIST подтверждены до 3 знаков |
| **scVelo** (Bergen 2020, Nat Biotech) | единичные клетки | ✅ **Конвейер воспроизведён** — выявлен баг ABI numpy 2.x, дававший 100% NaN, исправлен понижением до 1.26.4 |
| **Robust Stitching** (Ruiz 2023, ICML) | ML для изображений | ✅ **Конвейер воспроизведён** — устранено 5 поломок из-за устаревания (API torchvision, pandas `append`, отсутствующие зависимости) |
| **Annotatability** (Nitzan 2024, Nat Comp Sci) | единичные клетки | ✅ **Конвейер воспроизведён** — 6 раундов отладки API выявили отсутствующую зависимость `pooch` |
| **ScType** (Ianevski 2022, Nat Comms) | единичные клетки (R) | ✅ **Конвейер воспроизведён** — путь для R прошёл валидацию после понижения версии |
| **Cropformer** (Wang 2025, Plant Communications) | геномика культур | ⚠️ **Частично** — код, модель и обучение проверены, но репозиторий поставляет лишь 10 демо-образцов, поэтому PCC=0.92 из статьи невозможно воспроизвести в текущем виде |

**6 статей · 1 полное воспроизведение по всем метрикам · 4 воспроизведения конвейера · 1 честное частичное · 18 проверенных улучшений навыка**

> **Честный по замыслу.** UMAP остановился на 78.6% совпадения метрик — чуть *ниже* нашей планки «чистого воспроизведения» в 80% — и ReproRun сообщает об этом как о противоречии в данных, а не округляет вверх. Для Cropformer фреймворк работает от начала до конца, но опубликованным цифрам нужны реальные данные генома культур, которые репозиторий никогда не поставляет, — поэтому он помечен как Частично, а не Пройдено.

<details>
<summary><b>Разбор кейса — Cropformer (⚠️ Частичное воспроизведение)</b></summary>

**Проверено ✅**
- Репозиторий найден и клонирован (`jiekesen/Cropformer`; URL в статье был неверным)
- Окружение собрано — Python 3.10 + PyTorch 2.5.1 + CUDA 12.1
- Архитектура модели — Conv1d + 8-головное самовнимание (2.6M параметров)
- GPU-инференс на RTX 4090; предобученные веса загружаются и работают
- Цикл обучения сходится — loss 89,540 → 23,771

**Не удалось воспроизвести ❌**
- Метрики статьи (PCC=0.92, …) — репозиторий поставляет лишь 10 случайных демо-образцов, а не реальные данные культур
- Задача классификации — в `model_class.py` отсутствуют ключевые функции
- Вложенная кросс-валидация, отбор признаков MIC, кодирование SNP 0–9 — не реализованы в репозитории

**Первопричина:** публичный репозиторий только демонстрационный; полный конвейер (отбор MIC, вложенная CV, тюнинг Optuna) и реальные наборы данных не включены. Точное воспроизведение потребовало бы реальных данных генома культур → обработки PLINK → переписывания описанного конвейера (~1–2 недели работы с данными и вычислениями).
</details>

## 📦 Начало работы

ReproRun — это навык агента, работающий с любым совместимым AI-ассистентом для написания кода. Чтобы его использовать:

1. Используйте AI-агент для кода, способный загружать навыки.
2. Поместите папку `paper-reproduction/` туда, откуда ваш агент загружает навыки.
3. В сессии просто попросите — например, *«воспроизведи scVelo»* или *«воспроизведи Таблицу 2 из этой статьи»*. Навык срабатывает автоматически.
